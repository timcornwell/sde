Error handling: ERR routines

The error handling system consists of a small number of routines.

  • ERRREPOR (REASON, ROUTINE, TEXT): report an error to the error handling system. The error message is stored away and ERROR is set to be true.
  • ERRCHANG (REASON, ROUTINE, TEXT): changes an existing error which was reported in routine ROUTINE.
  • ERRREASO (REASON): gives the reason for the last reported error. If appropriate, an error can then be cancelled using ERRCANCE.
  • ERRCANCE: deletes all error conditions.
  • ERRTRACE (ROUTINE): is called to indicate that an error was reported by a subroutine called from this one.
