String handling: STR routines

String handling is central to SDE and so there are a number of useful routines. Remember that SDE names can look like Unix directories: e.g. 'VIS/OBS/I' for the Stokes I part of the observed visibility. We have routines to assemble such names.

  • STRLEN (STRING): returns the length of STRING, CHARACTER*(*) ignoring spaces.
  • STRAPPEN (STRING1, STRING2): is a subroutine which adds STRING2 to the end of STRING1 ignoring spaces. There also exist STRAPPE2, STRAPPE3 for multiple strings.
  • STRM2 (PART1, PART2): is a CHARACTER*(*) function which return PART1 and PART2 joined by a /. STRM3 and STRM4 also exist.
  • STREXTRA (LIST, N, PART): extracts the N'th part of the comma-delimited LIST, CHARACTER*(*) and returns it in PART, CHARACTER*(*).
  • STRINT(N): converts an integer to a string.
  • STRMATCH(LIST, STRING): is a LOGICAL function which is true if STRING is contained in LIST or if LIST = '*'.
  • STRSEARC (TARGET, LIST, N): is an INTEGER function which returns the index of a target TARGET, CHARACTER*(*) in an array of strings LIST(*), CHARACTER*(*). If there is no match then zero is returned.
