Quick Start

  • Find out where the home directory of sde is. For the AOC workstations it is /zia/u/sde. Depending upon your shell, either export or setenv the environment variable SDEROOT to this e.g.

    export SDEROOT=/zia/u/sde

    and then do:

    source $SDEROOT/sdeini.csh for the c shell

    . $SDEROOT/sdeini for the Bourne or Korn shell

  • SDE programs are run from unix command level. Each program looks after its own inputs, and saves them from run to run in any directory. To run a program just type the name at the UNIX prompt: e.g.

    uvmap

    The program will then start up and ask you for inputs. After setting the inputs you type go, and the program will run. You can then run it in the background and free up your terminal. To do this type Z and then bg to run it in the background. fg will retrieve it and run it in the foreground. For an emergency stop, or to stop clean after a satisfactory number of clean iterations, type C while the program is in the foreground. The unix command jobs will list your current processes, and ps v will tell you how much time is being using.

  • To get a list of available tasks, do sdetasks.
  • SDE tasks communicate with each other and with other packages via disk files, either FITS or internal SDE formats.
